### **SECTION 3: HOW — 솔루션 설명 (4~5 슬라이드)**

#### Slide 5: Dormammu는 무엇인가?

```
# Dormammu = 58개 세계를 동시에 탐색하는 에이전트 팀

> "단 하나의 질문으로, 에이전트 10명이 모든 가능성을 자동으로 평가하고
> 가장 흥미로운 이야기를 찾아냅니다."

## 이게 가능한 이유

1. **프롬프트가 코드** — Python 클래스 대신 Markdown 프롬프트로 에이전트 정의
2. **상태는 파일** — DB 없이 JSON + Markdown 파일만으로 모든 세계 기록
3. **평가가 자동** — 6차원 메트릭으로 각 세계의 품질 객관화
4. **선택이 명확** — 점수 기반으로 최고의 결말 자동 선택

**결과**: 새로운 창작 도메인도 프롬프트만 바꾸면 즉시 적용 가능!
```

#### Slide 8: 6차원 평가 메트릭

```
# 품질을 어떻게 자동화하는가?

## Dormammu의 "Braintrust" — 6차원 메트릭

| 차원 | 가중치 | 설명 |
|------|--------|------|
| **Character Fidelity** | 20% | 캐릭터 성격·동기 재현도 (Big-5 기반) |
| **Foreshadowing** | 20% | 복선 품질과 회수율 |
| **Narrative Flow** | 15% | 조상 노드와의 자연스러운 연결 |
| **Audience Resonance** | 15% | 독자가 흥미로워할 전개 |
| **Emergence** | 15% | 예상치 못한 창발적 사건 |
| **Plausibility** | 15% | 세계관 규칙 내 논리적 타당성 |

**점수 계산**:
- Composite = Σ(각 차원 × 가중치)
- **Score > 0.3 → EXPAND** (좋은 분기, 계속 탐색)
- **Score ≤ 0.3 → PRUNE** (약한 분기, 가지치기)

> "픽사의 Braintrust를 자동화했다."
```
